CloudStreams Configuration Screens : STS
STS
CloudStreams can act as a Security Token Service (STS) client. You can use CloudStreams’s default STS, DefaultSTS, by selecting this option on this screen. Alternatively, you can use a third-party STS that has been defined in the Integration Server (and described in the Web Services Developer's Guide, in the section Securing Web Services Using Policies Based on WS-SecurityPolicy).
To open the STS options screen, click Solutions > CloudStreams > Administration > STS in the Navigation panel.
To use a third-party STS, click Add new STS configuration on this screen and complete the Add New STS Configuration page as follows.
STS Configuration
Name
A unique name for the STS being configured. If this value is changed after creating an STS, the previous STS configuration will be deleted and replaced with the new one.
Endpoint
The STS endpoint to which the WS-Trust request will be sent by CloudStreams to obtain the SAML token.
Token Type
The type of token that CloudStreams should request from the STS. Value can be SAML_11 or SAML_20.
Note:  
Currently the default Integration Server STS only supports issuing SAML 2.0 Sender-vouches tokens.
WS-Trust Version
The version of WS-Trust that CloudStreams should use to send the RST to the SAML Issuer. Value can be VERSION_05_02 or VERSION_05_12.
Time-To-Live (TTL)
Indicates the time-to-live value in seconds that will be specified in the RST. If not specified, the default is 300 seconds (5 minutes).
Keystore
Select a keystore that has been configured in Integration Server. To configure a keystore in Integration Server, see the section Securing Communications with the Server in the webMethods Integration Server Administrator's Guide.
Sign Request/Signing Alias
Select Sign Request if the STS requires a signed request, and specify the signing alias in the Signing Alias field.
Encrypt Request/Encryption Alias
Select Encrypt Request if the STS requires an encrypted request, and specify the encryption alias in the Encryption Alias field.
HTTP Basic Authentication
If the STS requires authentication, enter the HTTP Basic Authentication Username and Password.
WS-Security Username Token
The WS-Security username token to send to the STS. Also specify the Username, username's Password, and Password Type (None, Text, or Digest).
Next, configure the desired virtual services so they can use the STS. To do this, write an IS service that includes the predefined Java service pub.cloudstreams.security.ws.AddSamlSenderVouchesToken and invoke this IS service in the "In Sequence" step of the virtual service. This service must reference an STS that is identified in CloudStreams. For details about the AddSamlSenderVouchesToken service, see the topic Using the Security API in IS Services in the document Administering webMethods CloudStreams.
Note:  
If you selected Integration Server’s default STS (DefaultSTS), edit the default STS's configuration file to specify the keystore and alias so that the STS can sign the SAML assertion that it is issuing. Use the file's comments as a guide to configure this file for your system. The configuration file is located here:
IntegrationServer_directory\config\security\saml\esb_sts.xml
