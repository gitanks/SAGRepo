Mediator Configuration Settings : Security Token Service (STS) Configuration
Security Token Service (STS) Configuration
Use the Security Token Service (STS) Configuration screen to configure Mediator to act as a Security Token Service (STS) client for SAML Sender-Vouches processing.
Note:  
This screen is valid for the services that have SAML policies enforced before version 9.12. For versions 9.12 and later, you can perform these configurations in CentraSite by selecting the SAML policies.
You can use the Integration Server’s default STS or you can use a third-party STS that has been defined in the Integration Server. The default STS supports only V2.0 SAML Sender-Vouches tokens.
For information about configuring other components (besides Mediator) of your system for SAML Sender-Vouches processing, see the section SAML Support in Mediator in the document Administering webMethods Mediator.
Click Add new STS configuration to configure a new STS. The configurable parameters are displayed.
Click DefaultSTS to edit Integration Server’s default STS. The configurable parameters are displayed.
The various parameters that can be configured are:
Configuration
Parameter
Description
Name
A unique name for the STS being configured. If this value is changed after creating an STS, the previous STS configuration will be deleted and replaced with the new one.
Endpoint
The STS endpoint to which the WS-Trust request will be sent by Mediator to obtain the SAML token.
Token Type
The type of the token that Mediator should request from the STS. The value can be SAML_11 or SAML_20. Note that currently the default Integration Server STS only supports issuing SAML 2.0 Sender-Vouches tokens.
WS-Trust Version
The version of WS-Trust that Mediator should use to send the RST to the SAML Issuer. The value can be VERSION_05_02 or VERSION_05_12.
Time To Live (TTL)
Indicates the time-to-live value in seconds that will be specified in the RST. If not specified, it is defaulted to 300 seconds (5 minutes).
KeyStore / Signing Alias / Encryption Alias
Displays the configured IS keystore names. If the STS requires a signed and/or encrypted request, you can configure the keystore to use and specify the signing alias and the encryption alias. Note that the signing alias and encryption alias drop-down boxes will be enabled only when the Sign Request and Encrypt Request check boxes are enabled.
HTTP Basic Authentication
Parameter
Description
Username
If the STS requires authentication, enter the HTTP Basic Auth username.
Password
If the STS requires authentication, enter the HTTP Basic Auth password.
WS-Security Username Token
Parameter
Description
Username
The WS-Security username token to send to the STS.
Password
The password of the WS-Security username token.
Password Type
The type of the password of the WS-Security username token.
The various operations performed in this screen are:
Command
Action
Return
Click to return to the previous screen.
Note:  
If you click return before saving, the modified configuration parameters will not be saved.
Save
Click to save the modified configuration parameters if any.
Note:  
If you selected Integration Server’s default STS (DefaultSTS), edit the default STS's configuration file to specify the keystore and alias so that the STS can sign the SAML assertion that it is issuing. Use the comments in the configuration file as a guide to configure this file for your system. The file is located here: Integration Server_directory\config\security\saml\esb_sts.xml. After you edit the file, re-start Integration Server. DefaultSTS is now ready to issue SAML tokens.
