Administering Mediator : EDA/Database Configuration
EDA/Database Configuration
Mediator can publish data about run-time events and key performance indicator (KPI) metrics:
to a JDBC destination. A JDBC connection pool is defined in Integration Server and associated with the Mediator functional alias. JDBC connection pools are specified in the Integration Server Administrator's Settings > JDBC Pools page.
If a JDBC connection pool is not defined in Integration Server, you must define one as described in the section Connect Products to Database Components in the document Installing Software AG Products.
Note:  
You cannot use Integration Server's embedded database for publishing events and metrics.
using the EDA capability of Universal Messaging to publish the event type(s) to the default endpoint configured in the universal messaging configuration.
Note:  
Ensure that you install and start the messaging server before you start Integration Server.
Alternatively, instead of using EDA/Database to publish the events and metrics, you can use other destinations such as, SNMP, Elasticsearch, API Portal, CentraSite, and so on.
The EDA/Database Configuration screen is used to specify:
The destination to which Mediator must publish the events.
The types of run-time events that Mediator must publish.
Whether the KPI metrics must be published.
Click Edit to edit the configuration parameters.
The various parameters that can be configured are:
Performance Metrics
Parameter
Description
Publish Interval (minutes)
Specify how often (in minutes) Mediator must report performance metrics. Enter a value from 1 through 60. The default is 60.
Note:  
You can only enter a value for this parameter if Report Performance Data is selected.
Event Types
Parameter
Description
Error
Select to publish Error events. An Error event occurs each time an invocation of a virtual service results in an error.
Lifecycle
Select to publish Lifecycle events. A Lifecyle event occurs each time Mediator starts or shuts down.
Policy Violation
Select to publish Policy Violation events. A Policy Violation event occurs each time an invocation of a virtual service violates a run-time policy that was set for the virtual service.
Destinations
Parameter
Description
Emit to Database
Specify whether Mediator must publish the events and KPI metrics to the JDBC database (that is, a JDBC connection pool associated with the function alias named, Mediator).
Emit to Default EDA Endpoint
Specify whether Mediator must publish the events and KPI metrics to the default EDA endpoint. If you select this option, Mediator publishes the events to the Universal Messaging event bus.
The various operations performed in this screen are:
Command
Action
Return
Click to return to the previous screen.
Note:  
If you click return before saving, the modified configuration parameters will not be saved.
Save
Click to save the modified configuration parameters if any.
