Administering Mediator : API Portal Configuration
API Portal Configuration
You can select API Portal as a destination from CentraSite. This enables Mediator to send data for run-time events and Key Performance Indicator (KPI) metrics, when the API is published from CentraSite to Mediator. The API Portal instances are displayed in Mediator once the API is published from CentraSite.
Click Edit to edit the configuration parameters.
The various parameters that can be configured are:
Event Types
Parameter
Description
Error
Select to publish Error events. An Error event occurs each time an invocation of a virtual service results in an error.
Lifecycle
Select to publish Lifecycle events. A Lifecyle event occurs each time Mediator starts or shuts down.
Policy Violation
Select to publish Policy Violation events. A Policy Violation event occurs each time an invocation of a virtual service violates a run-time policy that was set for the virtual service.
Performance Metrics
Parameter
Description
Publish Interval (minutes)
Specify how often (in minutes) Mediator must report performance metrics. Type a value from 1 through 60. The default is 60.
Note:  
You can only provide a value for this parameter if Report Performance Data is selected.
API Portal Instances
Note:  
The API Portal information is displayed when published from CentraSite.
Parameter
Description
Name
The name of the API Portal instance.
You can modify the following configuration parameters of an API Portal instance by selecting the name of the instance to be modified:
Name
URL
Tenant
User Name
Password
Retype Password
URL
The URL of the API Portal, for example,
http://hostname:portnumber
or
https://IP-address:portnumber
Tenant
An instance of API Portal.
User Name
The API Portal user name that Mediator must use to access API Portal.
If you are using the Operating System auth mechanism, use the following format for the user name: API-Portal-Host-Name\API-Portal-user-name.
Action
Click to delete an API Portal instance from Mediator. once the instance is republished in CentraSite, the instance will be visible in Mediator.
