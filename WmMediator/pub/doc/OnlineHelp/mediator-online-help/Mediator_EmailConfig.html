Administering Mediator : Email Configuration
Email Configuration
The Email Configuration screen is used to configure Mediator to send Transaction and Monitoring event notifications to an SMTP mail server as e-mail messages.
You can configure Mediator to:
Send monitoring alerts to an SMTP email server when user-specified performance conditions are violated. To do this, you can include the following actions in your virtual service's run-time policy:
Monitor Service Performance.
Monitor Service Level Agreement.
Throttling Traffic Optimization.
Log the payloads of all transactions to an SMTP email server, using the Log Invocations action. Mediator sends the payloads as email attachments that are compressed using gzip data compression.
Click Edit to edit the configuration parameters.
The various parameters that can be configured are:
Parameter
Description
SMTP Host Name/IP Address
The address of the SMTP server to be used by Mediator for sending email alerts.
Port
The host name or IP address of the machine on which CentraSite is running.
User
The user name of the email account used to log into your SMTP server.
Password
The password of the email account used to log into your SMTP server.
Retype Password
The same password you entered in the Password field.
From
The email address that will appear in the From field of the event email. If you leave this field blank, Mediator generates a default email address composed of the configured target name and hostname of the Integration Server, as follows:
Target-Name@hostname
TLS Enabled
Whether to use transport-layer security (TLS). If selected, the truststore configured for Mediator must include a certificate in the email server's certificate chain.
This requires that both a keystore and truststore be configured for Integration Server.
Note:  
You must also specify a value for the IS Truststore Name field in the Mediator Configuration screen (see General Configuration).
Test Recipient
The email address of the person who must receive the test email.
The various operations performed in this screen are:
Command
Action
Return
Click to return to the previous screen.
Note:  
If you click return before saving, the modified configuration parameters will not be saved.
Saving
Click to save the modified configuration parameters if any.
Test
Click to test your connection. If the connection is successful, Mediator sends an email to the address you entered in the Test Recipient field and displays a verification message.
